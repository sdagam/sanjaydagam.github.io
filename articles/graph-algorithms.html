Graph Algorithms: BFS, DFS, and Shortest Paths

#63 of 156 March 15, 2025 Group 3 · Jan–Jun 2025

How does Google Maps find the fastest route across a city? How does Facebook suggest friends you might know? How does a web crawler discover new pages? These questions all reduce to searching and navigating graphs—mathematical structures of nodes connected by edges. Graph algorithms are among the most practically important algorithms in computer science, providing systematic methods for traversal, shortest paths, connectivity, and flow that power everything from navigation to network analysis to compiler design.

Breadth-First Search

BFS explores a graph level by level outward from a source node. Using a queue, BFS visits all neighbors of the source before any of their neighbors. This guarantees that when a node is first visited, it has been reached by the shortest path (fewest edges) from the source. BFS runs in O(V + E) time (V vertices, E edges) and simultaneously computes: the shortest path in terms of hops from source to every reachable node, the connected component containing the source, and whether the graph is bipartite (2-colorable). BFS underlies social network distance calculations, peer-to-peer network routing, and finding the shortest word ladder between two words.

BFS time: O(V + E) Guarantees shortest path (by edge count) from source to all reachable nodes

Depth-First Search

DFS explores as far as possible along each branch before backtracking, using a stack (or recursion). DFS runs in O(V + E) and produces a DFS tree with discovery and finish timestamps for each node. These timestamps reveal graph structure: edges not in the DFS tree fall into three categories (back, forward, cross edges) that characterize cycles, DAGs, and strongly connected components. DFS finds topological orderings of DAGs (valid task schedules respecting dependencies), detects cycles, and is the basis for Tarjan's and Kosaraju's algorithms for finding strongly connected components in directed graphs.

Dijkstra's Algorithm

For weighted graphs where edge weights represent distances or travel times, BFS's hop count is insufficient. Dijkstra's algorithm finds shortest weighted paths from a source using a greedy strategy: always extend the currently shortest known path. Using a min-heap, it runs in O((V + E) log V) time. The algorithm maintains a distance array (initialized to ∞ except source = 0), extracts the minimum-distance unvisited node, and relaxes all its outgoing edges—updating distances if a shorter path is found. Dijkstra requires non-negative edge weights; for graphs with negative weights, Bellman-Ford is needed.

Dijkstra relaxation: dist[v] = min(dist[v], dist[u] + w(u,v)) Time: O((V+E) log V) with binary heap

A* Search

A* extends Dijkstra with a heuristic h(v) estimating the remaining distance from node v to the target. Instead of the minimum dist[v] node, A* extracts the minimum of dist[v] + h(v)—combining known cost with estimated future cost. With an admissible heuristic (never overestimates true remaining cost), A* is guaranteed to find the optimal path while exploring far fewer nodes than Dijkstra. Google Maps uses A* (or variants) with geographic distance as the heuristic for routing. The same framework drives pathfinding in video games, robot motion planning, and puzzle solvers.

A* priority: f(v) = g(v) + h(v) where g = cost so far, h = heuristic to goal Optimal if h is admissible: h(v) ≤ true remaining distance

Maximum Flow

The maximum flow problem asks: given a directed graph with edge capacities, what is the maximum amount of flow that can be sent from source to sink? The Ford-Fulkerson method repeatedly finds augmenting paths from source to sink and pushes flow along them until no augmenting path exists. The max-flow min-cut theorem is a fundamental duality result: maximum flow equals minimum cut (the minimum capacity edge set whose removal disconnects source from sink). Applications include network routing capacity, bipartite matching (job assignments, kidney transplant pairing), image segmentation, and airline scheduling.

Conclusion

Graph algorithms translate the abstract mathematics of graphs into practical computational tools. BFS and DFS are the foundation—simple traversals that reveal connectivity, structure, and paths. Dijkstra and A* optimize traversal for weighted networks. Max-flow solves resource routing. Together, they form the computational backbone for navigation, network analysis, scheduling, and countless other applications where the world's connections can be modeled as graphs. Mastering these algorithms means acquiring tools applicable across virtually every domain of computer science and engineering.
