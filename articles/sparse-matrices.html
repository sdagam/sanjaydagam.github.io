Sparse Matrices and Graphs: Efficient Computation at Scale

#120 of 156 April 18, 2026 Group 5 · Jan–Jun 2026

Google's PageRank algorithm computes a score for every one of the roughly 100 billion pages on the web. To do this, it must work with a matrix that is 100 billion by 100 billion — containing 10²² entries. Storing a full matrix this size would require more storage than all the world's computers combined. Yet Google runs PageRank in hours. The secret: almost all entries of the matrix are zero. A web page links to maybe 50 other pages out of 100 billion. The matrix is 99.9999999% zeros — it is sparse. And sparse matrices can be stored and computed with using only the nonzero entries.

What Is a Sparse Matrix?

A dense matrix stores every entry — including all the zeros. A sparse matrix stores only the nonzero entries and their positions. The savings can be astronomical. Google's web link matrix has about 100 billion nonzero entries (one per link). A dense matrix would need 10²² entries — 100 billion times larger. Sparse storage keeps only what matters.

Dense storage: store all n² entries of an n×n matrix Sparse storage: store only the k nonzero entries (their values and positions) For Google's matrix: n = 100 billion, k ≈ 100 billion Dense: n² = 10²² entries needed (impossible) Sparse: k = 10¹¹ entries needed (manageable) Memory saved: factor of n²/k = 10²²/10¹¹ = 10¹¹ (100 billion times less!)

The Connection to Graphs

Every graph (network) corresponds to a sparse matrix — the adjacency matrix — where entry (i,j) is 1 if node i connects to node j, and 0 otherwise. Real networks are always sparse: the internet has billions of nodes but each node connects to only a handful of others. A social network might have a billion users but even a very popular person follows only a few million — still a tiny fraction of a billion. The graph structure guarantees sparsity, and sparsity makes computation feasible.

Sparse Matrix-Vector Multiplication

The most common operation in scientific computing is multiplying a matrix by a vector: compute Ax. For a dense n×n matrix, this requires n² multiplications. For a sparse matrix with k nonzero entries, it requires only k multiplications — one per nonzero entry. PageRank is computed by repeatedly multiplying the sparse web matrix by a vector, one multiplication per iteration. Each multiplication takes time proportional to k (100 billion operations) rather than n² (10²²). That's the difference between hours and the age of the universe.

Dense Ax: n² multiplications and additions Sparse Ax: k multiplications and additions (k << n²) For PageRank: Dense: 10²² operations per iteration (impossible) Sparse: 10¹¹ operations per iteration (feasible in hours)

Sparse Solvers: Systems of Equations

Beyond multiplication, sparse methods also solve systems of equations Ax = b efficiently. Standard Gaussian elimination on a dense n×n matrix takes n³ operations — for n = 1 million, that's 10¹⁸, impossibly slow. Sparse direct solvers exploit the zero structure to eliminate much of this work. Iterative sparse solvers (like conjugate gradient) solve Ax = b by repeated sparse matrix-vector multiplications, converging in far fewer than n steps for well-conditioned problems.

These sparse solvers are what make finite element analysis (Article #64) practical. A structural simulation might have a million nodes, each connected to a handful of neighbors. The resulting million-by-million stiffness matrix is extremely sparse. Dense methods would be impossible; sparse methods make the solution run in minutes on a laptop.

Other Applications

Sparse matrices power GPS navigation (solving large sparse systems from satellite signals), climate models (discretizing the atmosphere into millions of grid cells, each coupled only to neighbors), medical imaging reconstruction (sparse tomography matrices), and chip design (analyzing circuits with millions of components connected in sparse networks). Machine learning on graph-structured data — social networks, molecular structures, citation graphs — uses sparse matrix operations throughout.

Conclusion

Sparse matrices are the key to making linear algebra work at real-world scale. When most entries are zero — as they always are in large networks, physical simulations, and scientific models — storing and computing with only the nonzero entries transforms impossible problems into tractable ones. The connection to graphs is fundamental: any network gives a sparse matrix, and any sparse matrix encodes a network. Google's PageRank, finite element simulations, and GPS positioning all depend on the same insight: ignore the zeros, work with what's there.
