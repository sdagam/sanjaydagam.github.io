Neural Networks: Layers, Weights, and Activation Functions

#68 of 156 April 19, 2025 Group 3 · Jan–Jun 2025

A neural network that can recognize any photograph, translate between languages, write code, and hold a conversation was trained on data using an algorithm that adjusts millions of numbers—weights—to minimize prediction errors. No explicit rules were programmed; the network learned statistical patterns from examples. Understanding how neural networks represent, learn, and generalize is essential for anyone working with modern AI—and it requires understanding the linear algebra, calculus, and optimization that make them work.

Architecture: Layers and Neurons

A feedforward neural network organizes computation into layers. Each neuron in a layer computes a weighted sum of its inputs and applies a nonlinear activation function: output = f(Σ wᵢxᵢ + b), where wᵢ are weights, b is a bias, and f is the activation. In matrix form, one layer computes h = f(Wx + b), where W is the weight matrix and f is applied elementwise. Stacking multiple layers creates a deep network: h₁ = f₁(W₁x + b₁), h₂ = f₂(W₂h₁ + b₂), and so on. The depth enables hierarchical representations: early layers detect simple features; later layers combine them into complex concepts.

Single layer: h = f(Wx + b) Deep network: y = fₙ(Wₙ fₙ₋₁(… f₁(W₁x+b₁) … ) + bₙ)

Activation Functions

The activation function introduces nonlinearity essential for universal approximation. ReLU (Rectified Linear Unit): f(x) = max(0,x)—simple, cheap to compute, and solves the vanishing gradient problem that plagued sigmoid and tanh activations in deep networks. Sigmoid: f(x) = 1/(1+e^{−x})—used in output layers for binary classification, squashing output to (0,1). Softmax: f(xᵢ) = e^{xᵢ}/Σe^{xⱼ}—used for multi-class classification, converting raw scores to probabilities summing to 1. GELU (Gaussian Error Linear Unit) is preferred in Transformers. The choice of activation function significantly affects training dynamics and performance.

ReLU: f(x) = max(0,x) Sigmoid: f(x) = 1/(1+e⁻ˣ) Softmax: f(xᵢ) = eˣⁱ/Σeˣʲ

Backpropagation

Neural networks are trained by minimizing a loss function via gradient descent. Backpropagation efficiently computes gradients of the loss with respect to all weights using the chain rule applied layer by layer. Starting from the output loss, gradients flow backward: the gradient of the loss with respect to layer k's weights is computed from the gradient with respect to layer k+1's activations. This reverse-mode automatic differentiation requires only one forward pass and one backward pass—O(n) in the number of parameters—far cheaper than finite-difference approximation. Backpropagation is the reason deep learning is computationally feasible.

Backprop chain rule: ∂L/∂Wₖ = (∂L/∂hₖ) · (∂hₖ/∂Wₖ) Each layer's gradient uses the gradient from the layer above

Universal Approximation

The Universal Approximation Theorem guarantees that a feedforward network with a single hidden layer of sufficient width can approximate any continuous function on a compact domain to arbitrary precision. This ensures neural networks are expressively powerful enough for any task—in principle. In practice, depth is crucial: deep narrow networks often achieve better generalization than shallow wide networks with the same parameter count, and many practical architectures (CNNs for images, Transformers for sequences) exploit structural inductive biases that shallow networks lack.

Regularization and Generalization

A network with enough parameters can memorize training data perfectly while failing on new examples—overfitting. Regularization techniques prevent this. L2 regularization (weight decay) penalizes large weights, keeping the network smooth. Dropout randomly zeros neurons during training, forcing redundant representations. Batch normalization normalizes activations within each mini-batch, stabilizing training and allowing higher learning rates. Data augmentation (random crops, flips, color jitter) artificially expands the training set. These techniques collectively enable networks to generalize—learning patterns rather than memorizing examples.

Conclusion

Neural networks combine linear algebra (weight matrices), calculus (backpropagation), and optimization (gradient descent) into a learning system of remarkable power. Their success comes not from mimicking biological neurons but from being flexible, differentiable function approximators trainable end-to-end from data. Understanding the mathematics—how layers compose, how gradients flow backward, what activations do—transforms neural networks from mysterious black boxes into interpretable computational systems whose behavior can be reasoned about, improved, and adapted to new problems.
