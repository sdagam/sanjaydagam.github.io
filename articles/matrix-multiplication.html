Matrix Multiplication: Composing Linear Transformations

#82 of 156 July 26, 2025 Group 4 · Jul–Dec 2025

Every frame of a Pixar movie involves rotating, scaling, and repositioning thousands of 3D objects — characters, props, lights — and rendering them onto a flat screen. The animators don't do this by hand for every object in every frame. Instead, each transformation is encoded as a matrix, and combining transformations is done by multiplying those matrices together. Understanding why that works is the key to understanding matrix multiplication.

A Matrix Is a Transformation

Start with one idea: a matrix is a recipe for moving vectors around. Multiply a matrix by a vector and you get a new vector — the original, transformed. A rotation matrix rotates every vector by a fixed angle. A scaling matrix stretches or shrinks. A reflection matrix flips across an axis. In 3D animation, every object in the scene has a position described by vectors, and moving that object means multiplying by transformation matrices.

Combining Two Moves

Here's where multiplication comes in. Suppose you want to rotate a character, then move it to the right. That's two transformations in sequence. You could apply the rotation matrix first, then the translation matrix. Or — and this is the key insight — you could multiply the two matrices together first to get a single combined matrix, then apply that to every point in the character. One matrix, one multiplication per point, same result.

Apply A then B to vector v: B(Av) = (BA)v Combined matrix: C = BA → Cv gives the same result

Notice the order: B comes before A in the product even though A is applied first. This trips people up, but it makes sense once you see it as function composition — just like f(g(x)) means "do g first, then f."

How the Arithmetic Works

The (i, j) entry of the product AB is computed by taking row i of A and column j of B, multiplying the matching entries, and adding them all up. That's it. The reason for this specific recipe — and not some simpler one, like just multiplying matching entries — is that it's the only definition that makes the composition interpretation work out correctly. The formula is engineered to be geometrically meaningful.

(AB)ᵢⱼ = (row i of A) · (column j of B)

Order Matters — Always

One thing that surprises people: AB and BA are generally different matrices. Rotating then scaling gives a different result than scaling then rotating. In animation, this matters enormously — the order in which you apply a character's joint rotations determines the final pose. A shoulder rotation followed by an elbow rotation produces a very different arm position than the same rotations in reverse order. Matrix multiplication captures this real asymmetry in the world.

Other Places This Shows Up

3D animation is just one application. Every time your phone renders graphics, it's multiplying transformation matrices. Robotics uses matrix chains to compute a robot arm's position from its joint angles. Quantum mechanics uses matrix multiplication to track how particles evolve over time. The same operation — rows times columns — underlies all of these, because in each case you're composing one transformation with another.

Conclusion

Matrix multiplication isn't an arbitrary arithmetic rule — it's the mathematically correct way to combine two transformations into one. When Pixar rotates then scales a character, they're computing a matrix product. When your phone maps 3D geometry to a 2D screen, it's multiplying transformation matrices. Understanding multiplication as composition turns a confusing procedure into an obvious necessity: of course you do it that way, because that's what combining transformations means.
