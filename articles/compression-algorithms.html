Compression Algorithms: Storing More with Less

#73 of 156 May 24, 2025 Group 3 · Jan–Jun 2025

A high-resolution photograph contains 24 megabytes of raw data. Stored as a JPEG, it takes perhaps 2 megabytes—a 12:1 compression ratio—with barely perceptible quality loss. A one-hour HD video contains 50 gigabytes raw; compressed to H.264, it fits on a DVD. Without compression, the internet as we know it would be impossible: video streaming, video calls, and image sharing would consume bandwidth and storage orders of magnitude beyond what exists. Compression algorithms combine information theory, signal processing, and clever data structures to represent information in far fewer bits.

Lossless Compression: Entropy Coding

Lossless compression removes statistical redundancy without discarding any information. Huffman coding assigns shorter bit sequences to more frequent symbols: the letter 'e' (12.7% frequency in English) gets a short code like '00'; 'z' (0.07%) gets a long code like '11011010'. The average code length approaches the Shannon entropy H = −Σ pᵢ log₂ pᵢ—the theoretical minimum bits per symbol. Arithmetic coding goes further, encoding entire messages as a single fraction, achieving entropy exactly. LZ77 and LZ78 (used in ZIP and gzip) find repeated patterns and replace them with references to earlier occurrences—exploiting long-range statistical structure beyond single-symbol frequencies.

Shannon entropy: H = −Σ pᵢ log₂ pᵢ (minimum bits per symbol) Huffman coding achieves average length ≤ H + 1 bit/symbol

Lossy Compression: The JPEG Story

JPEG compresses images by discarding information human vision barely notices. The algorithm: divide the image into 8×8 pixel blocks; apply the Discrete Cosine Transform (DCT) to represent each block as a sum of frequency components; quantize the DCT coefficients (round them, discarding small high-frequency components—the compression step); entropy code the remaining nonzero coefficients. The DCT concentrates energy in low-frequency components; quantization zeros out high-frequency components (fine textures, sharp details) that the eye barely perceives. The quality parameter controls how aggressively to quantize—higher quality means less quantization and larger files.

JPEG: image block → DCT → Quantize (lossy step) → Entropy code DCT coefficient: F(u,v) = (2/N) Σ Σ f(x,y)·cos(πu(2x+1)/2N)·cos(πv(2y+1)/2N)

Video Compression: Exploiting Time

Video compression exploits temporal redundancy: consecutive frames are nearly identical, differing only in moving objects. MPEG standards divide frames into I-frames (complete images, compressed like JPEG), P-frames (predicted from the previous frame—only differences are stored), and B-frames (predicted from both previous and next frames). Motion estimation finds how blocks of pixels moved between frames; motion compensation stores the motion vectors plus the prediction residuals. This temporal prediction reduces the data needed by factors of 10–100 compared to compressing each frame independently, enabling HD video at manageable bitrates.

Video compression: store motion vectors + residuals instead of full frames P-frame data ≈ 10-20× less than I-frame for typical content

Neural Compression

Recent neural network-based compression approaches achieve better quality at lower bitrates than classical algorithms for images, audio, and video. Variational autoencoders (VAEs) learn compact latent representations optimizing a rate-distortion tradeoff: minimize bits used while minimizing perceptual distortion. The encoder maps images to a compressed latent code; the decoder reconstructs from it. Neural compression can capture semantic structure—compressing a face image by representing facial geometry and texture separately—rather than treating images as generic signal. HEVC and AV1 video codecs incorporate neural network components, and fully neural codecs are advancing rapidly.

Rate-Distortion Theory

Shannon's rate-distortion theory establishes fundamental limits on lossy compression: for a given acceptable distortion level D, there is a minimum bit rate R(D) below which it is impossible to compress while staying within the distortion bound. The rate-distortion curve R(D) decreases as D increases—tolerating more distortion allows fewer bits. This theoretical framework guides practical algorithm design, providing benchmarks against which codecs are measured. Modern image and video codecs typically operate within 2–5× of the Shannon limit, meaning improvements must come from better perceptual models rather than purely information-theoretic efficiency.

Rate-distortion: R(D) = min_{p(x̂|x): E[d(X,X̂)]≤D} I(X; X̂) (minimum mutual information for given distortion)

Conclusion

Compression algorithms are applied information theory—taking Shannon's abstract measure of information and turning it into algorithms that shrank the internet's storage and bandwidth requirements by orders of magnitude. Lossless compression removes statistical redundancy; lossy compression discards what humans won't notice. Neural approaches are pushing the frontier toward the rate-distortion theoretical limits. Every image you view, video you stream, and audio file you play is the output of these mathematical techniques, silently making the modern information infrastructure possible.
