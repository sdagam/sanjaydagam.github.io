Dynamic Programming: Solving Complex Problems by Breaking Them Down

#53 of 156 January 4, 2025 Group 3 · Jan–Jun 2025

How do you find the shortest route through a city with thousands of intersections? How do you determine the best sequence of investments over a decade? How do you align two DNA sequences to find mutations? These problems share a hidden structure: they can be broken into overlapping subproblems whose solutions can be reused. Dynamic programming—coined by Richard Bellman in the 1950s—transforms exponential-time brute-force searches into efficient polynomial-time algorithms by systematically storing and reusing partial results.

The Core Idea: Overlapping Subproblems

Consider computing the nth Fibonacci number naively: F(n) = F(n−1) + F(n−2). A recursive implementation recomputes F(2) an exponential number of times. Dynamic programming avoids this by storing each computed value—memoization (top-down) stores results as they're computed; tabulation (bottom-up) fills a table from small to large. Either way, each subproblem is solved exactly once. The Fibonacci computation drops from O(2ⁿ) to O(n). This pattern—identify overlapping subproblems, store their solutions, build larger solutions from smaller ones—is the DP template.

Naive Fibonacci: O(2ⁿ) time DP Fibonacci: O(n) time, O(n) space (or O(1) with rolling variables)

Optimal Substructure

DP also requires optimal substructure: an optimal solution to the full problem contains optimal solutions to its subproblems. The shortest path from A to C through B consists of the shortest path from A to B plus the shortest path from B to C. This allows building optimal global solutions from optimal local ones. Problems lacking this property—like the longest simple path in a graph with cycles—cannot be solved by DP because the best subpath might not be part of the best full path. Identifying whether a problem has both properties is the first step in applying DP.

Classic Examples

The 0/1 knapsack problem: given items with weights and values, maximize value subject to a weight capacity. The DP table has rows for items and columns for capacities; each cell records the best value achievable with the first i items and capacity j. Longest common subsequence (used in diff tools and DNA alignment): fill an (m+1)×(n+1) table where each cell records the LCS length of the first i characters of one string and first j of the other. Matrix chain multiplication: find the order to multiply matrices minimizing operations—a classic O(n³) DP that underlies compiler optimization.

Knapsack recurrence: dp[i][w] = max(dp[i−1][w], dp[i−1][w−wᵢ] + vᵢ)

Bellman-Ford and Shortest Paths

Bellman's original application was optimal control and shortest paths. The Bellman-Ford algorithm finds shortest paths in graphs with negative edge weights by relaxing all edges n−1 times: if the path to v through u is shorter than the known path to v, update it. This DP on path length handles negative weights that Dijkstra's algorithm cannot. The Floyd-Warshall algorithm extends this to all-pairs shortest paths in O(n³): dp[i][j][k] is the shortest path from i to j using only nodes 1 through k as intermediates, building up from k=0 to k=n.

Bellman-Ford: dist[v] = min(dist[v], dist[u] + w(u,v)) for all edges (u,v), repeated n−1 times

DP in Modern ML

Dynamic programming underlies key algorithms in machine learning and NLP. The Viterbi algorithm for Hidden Markov Models—used in speech recognition and gene finding—uses DP to find the most likely sequence of hidden states. The CYK algorithm parses context-free grammars in O(n³) using DP. Sequence-to-sequence models in neural machine translation use beam search, a heuristic DP variant. Reinforcement learning's value iteration and policy iteration are DP algorithms on Markov Decision Processes, computing optimal policies by solving Bellman equations iteratively.

Conclusion

Dynamic programming is a paradigm shift in problem-solving: from trying all possibilities to systematically building solutions from their components. The combination of overlapping subproblems and optimal substructure turns intractable combinatorial searches into efficient algorithms. Recognizing these properties in a new problem is the key skill—once identified, the DP table design and recurrence usually follow naturally, transforming an exponential problem into a polynomial one through organized reuse of work.
