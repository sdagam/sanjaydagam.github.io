Bayesian Networks: Modeling Probabilistic Dependencies

#70 of 156 May 3, 2025 Group 3 · Jan–Jun 2025

A patient has a fever and cough. Given these symptoms, what is the probability of pneumonia versus influenza versus COVID-19? Answering this requires modeling probabilistic dependencies: fever and cough are both caused by respiratory infections, so they're dependent; knowing the infection type renders them conditionally independent of each other given the diagnosis. Bayesian networks—directed graphs encoding probabilistic dependencies through conditional probability tables—provide a principled framework for reasoning under uncertainty, combining domain knowledge with data in a transparent, interpretable structure.

Structure: The DAG

A Bayesian network consists of a directed acyclic graph (DAG) and a set of conditional probability distributions. Each node represents a random variable. Directed edges represent direct probabilistic influences—a parent node directly influences its child. Each node has a conditional probability table (CPT): P(node | its parents). For binary variables, a node with k parents has a CPT with 2^k rows. The joint probability of all variables factors as: P(X₁, …, Xₙ) = Π P(Xᵢ | parents(Xᵢ)). This factorization—the key property—allows joint distributions over many variables to be represented compactly by specifying only local dependencies.

Joint distribution: P(X₁,…,Xₙ) = Πᵢ P(Xᵢ | Parents(Xᵢ)) Conditional independence: Xᵢ ⊥ non-descendants | Parents(Xᵢ)

Conditional Independence

The graph structure encodes conditional independence: a node is independent of its non-descendants given its parents. The d-separation criterion determines, from graph structure alone, whether any two sets of variables are conditionally independent given a third. Three fundamental patterns: common cause (X ← Z → Y: X and Y are independent given Z), common effect (X → Z ← Y: X and Y become dependent given Z—explaining away), and chain (X → Z → Y: X and Y are independent given Z). These patterns—especially explaining away, where observing an effect makes its causes compete—capture subtle probabilistic reasoning that humans find difficult but Bayesian networks handle automatically.

Inference

Given a Bayesian network, inference computes posterior probabilities of variables given observed evidence. Exact inference using variable elimination runs in time exponential in the treewidth of the graph—efficient for tree-structured networks, intractable for dense graphs. Approximate methods like belief propagation (exact on trees, approximate with loops) and Monte Carlo sampling (MCMC) scale to large networks. Medical diagnosis systems, fault detection in industrial equipment, and spam filtering all use Bayesian network inference to combine evidence from multiple uncertain sources into posterior probability estimates.

Learning from Data

Bayesian network parameters (the CPTs) can be estimated from data by maximum likelihood—simply counting conditional frequencies. Structure learning—inferring the DAG itself from data—is an NP-hard combinatorial problem, typically addressed through score-based methods (searching for the structure maximizing a penalized likelihood score like BIC) or constraint-based methods (testing for conditional independence and using the results to construct a compatible DAG). Learning structure from data faces fundamental identifiability limits: many different DAGs encode the same conditional independence structure and cannot be distinguished by data alone.

Dynamic Bayesian Networks

Dynamic Bayesian networks (DBNs) extend static networks to temporal data by modeling how variables evolve over time. The Hidden Markov Model (HMM) is a simple DBN where a hidden state at each time step directly generates an observation and transitions to the next hidden state. The Kalman filter—widely used in navigation and tracking—is a DBN with continuous Gaussian variables, enabling exact inference in linear systems. DBNs model gene regulatory networks, speech recognition, and robot localization, combining Bayesian network structure with temporal dynamics.

Conclusion

Bayesian networks provide a graphical language for representing, reasoning about, and learning from uncertain, interdependent information. Their power lies in making probabilistic structure explicit and interpretable—domain experts can encode knowledge as graph structure, and inference algorithms efficiently compute consequences. The combination of principled uncertainty quantification, transparent structure, and learnable parameters from data makes Bayesian networks indispensable tools wherever complex systems must be diagnosed, predicted, or controlled under uncertainty.
